Example - 프로젝터

  • 프로젝터는 블랙박스이다

  • 어떻게 동작하는지 알 필요는 없음

  • 인터페이스만 알면 됨: 입력/출력

  • 해당 입력과 통신할 수 있는 전자장치를 연결

  • 블랙박스는 어떻게든 입력 소스의 이미지를 변환하여 확대하고 벽에 분사

  • ABSTRACTION IDEA: 프로젝트를 사용하기 위해 프로젝터가 어떻게 동작하는지를 알 필요는 없음

Example - Module로서의 프로젝터

  • 올림픽을 위한 대형 이미지를 별도의 프로젝터에 대한 작업으로 분해하여 투사

  • 각 프로젝터는 입력을 받아 별도의 출력을 생성

  • 모든 프로젝터가 함께 작동하여 더 큰 이미지를 생성

  • DECOMPOSITION IDEA: 다양한 장치가 함께 작동하여 최종 목표를 달성

Decomposition(분해)를 사용하여 구조 만들기

  • 프로젝터 예제는, 장치를 분할하는 것

  • 프로그래밍에서는, 코드를 module(모듈)로 분해

    • 자체 포함된(self-contained)

    • 코드 분할(break up)에 사용

    • 재사용(Reusable)을 목적

    • 코드를 정리(organized)하는데 사용

    • 코드를 일관되게(coherent) 유지

  • 이 강의에서는 함수를 사용하여 분해를 달성하고

  • 후에 클래스를 사용하여 분해를 달성

Abstraction으로 세부 사항을 숨김

  • 프로젝터 예에서는 사용 방법에 대한 지침만 있으면 충분, 프로젝터의 제작 방법을 알 필요는 없음

  • 프로그래밍에서는 코드 조각을 블랙박스로 생각

    • 세부 정보를 볼 수 없음

    • 세부 정보를 볼 필요가 없음

    • 세부 정보를 보고싶지 않음

    • 지겨운 코드 세부사항을 숨김

  • Function specification(함수 명세)나 docstring(문서 문자열)으로 abstraction 달성

Function

  • 재사용 가능한 코드의 조각/덩어리로, function(함수)라고 부름

  • 함수는 직접 호출(call) 또는 간접 호출(invoke)되기 전까지는 프로그램에서 실행되지 않음

  • 함수의 특징:

    • 이름이 있음

    • parameter를 가짐(없거나, 또는 여러개)

    • docstring을 가짐(옵션이지만 권장됨)

    • body를 가짐

    • 무엇인가를 return

함수를 작성하고, Call/Invode 하는 방법 (확인 후 추가 작성 필요)

def is_even( i ): # def - keyword, is_even - name, i - parameter
    """   # specification, docstring
    input: i, a positive int
    Returns True if i is even, otherwise False
    """
    print("inside is_even") # body 시작
    return i % 2 == 0       # body 끝

is_even(3) # 함수 작성 후에, 함수의 이름과 파라미터 값을 사용하여 호출

Function Body

def is_even(i):
    """
    input: i, a positive int
    Returns True if i is even, otherwise False
    """
    print("inside is_even") # 명령 호출
    return i % 2 == 0       # 식으로 값을 계산하고, 값을 반환

Variable Scope

  • formal parameter(형식 파라미터)는 함수가 호출될 때, actual parameter(실제 파라미터) 값에 바인딩 됨

  • 함수로 진입할 때 새로운 scope/frame/environment(범위/프레임/환경) 생성

  • 범위는 이름을 객체에 매핑하는 것

    def f( x ):
    x = x + 1
    print('in f(x): x=', x)
    return x

    함수 정의

    • f(x)의 정의된 x는 formal parameter

    x = 3
z = f( x )

    Main프로그램 코드

    • x 변수 값 초기화

    • f(x)로 함수 호출

      • f()에 주어지는 x는 actual parameter

    • 함수의 return 값을 변수 z에 할당

return vs. print

return

print

  • return은 함수 내부에서 동작

  • 함수 내부에서 실행된 하나의 값만 return

  • 함수 내부의 코드지만 return문 이후에는 실행되지 않음

  • 함수 호출자에게 제공되는 값

  • print는 함수 외부에서 사용

  • 함수 내부에서 많은 print가 실행될 수 있음

  • 함수 내부의 코드로 print후의 코드도 실행됨

  • 연관된 값들을 콘솔에 출력함

Argument로서의 Function

def func_a():
    print('inside func_a')

def func_b(y):
    print('inside func_b)')
    return y

def func_c(z):
    print('inside func_c')
    return z()

print(func_a())       # func_a 호출, 파라미터 없음
print(5 + func_b(2))  # func_b 호출, 하나의 파라미터
print(func_c(func_a)) # func_c 호출, 하나의 파라미터로 다른 함수

Scope Example

  • 함수 내부에서, 외부에 선언된 변수에 접근 가능

  • 함수 내부에서, 외부에 선언된 변수의 값을 변경할 수 없음 - global variable(전역 변수)를 사용할 수 있으나 권장하지 않음

def f(y):
    x = 1  # 설명 1
    x += 1
    print(x)

x = 5      # 설명 2
f(x)
print(x)
def g(y):
    print(x)     # 설명 1
    print(x + 1)

x = 5            # 설명 2
g(x)
print(x)
def h(y):
    x += 1       # 설명 1

x = 5
h(x)
print(x)

  • 설명

    1. f()에서 정의된 지역 변수

    2. f()에서 정의된 지역 변수과 다른 전역 변수 x

  • 설명

    1. g() 외부에서 정의된 전역 변수 x

    2. g()에서 사용되었던 전역 변수 x

  • 설명

    1. UnboundLocalError: 지역 변수 x가 할당되기 전에 참조됨
